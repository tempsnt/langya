========================================
person
存放 人物类 从皇帝 到街头平民 各有各的性格 BASE里存放各种基类
========================================
role
存放 角色类 即同一个人物 可以被塞进不同的角色 然后就具有不同角色的特性
========================================
relation
存放 人物关系类 是一张图 记录本局有那些人物 人物之间有那些联系
========================================
object
存放 物品类 带有特殊信息的证物
========================================
event
存放 事件类 该类比较复杂 有各种流程 或有分支 会产生各种消息体 消息发送给各个角色之后会产生各种效果 
========================================
effect
存放 消息影响类 该类比较复杂 即不同的人物得到不同消息后 对人物关系类产生的影响 根据性格影响会变化很大
========================================
message
存放 消息类 消息一般是事件运行过程中发出的 也可单独发出 传播到各个人物的速度不同 人物的反映通过EFFECT类实现
========================================
actoin
存放 行动类 NPC 根据人物关系图数值的变化 产生各种行动 玩家可主动行动 
========================================
character
存放 性格类 NPC 宽容度 等等 需要仔细设计才行




每个类都有从数据库读取参数的能力
